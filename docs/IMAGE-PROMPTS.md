# Freshdocs Image Prompts

These prompts keep Freshdocs visuals reproducible without tying the README to one generator.

## README Hero

Inspired by the local image prompt archive and the CodeDesigner/UniversalUI reference scan. The useful pattern was not a specific asset; it was the structure: use a clear asset type, subject, style, composition, palette, constraints, and an explicit no-text rule.

Prompt used for `assets/freshdocs-hero.png`:

```text
Use case: productivity-visual
Asset type: GitHub README hero image and social visual for an open-source developer CLI
Primary request: Create a polished 16:9 product illustration for Freshdocs, a local documentation cache for coding agents.
Scene/backdrop: a clean dark developer workstation view with a local database node, official documentation pages, and an agent terminal connected by luminous lines.
Subject: source documents flowing into a local cache, then into a compact agent context block, with small trust symbols like check marks, shield shapes, and version pins.
Style/medium: premium semi-realistic 3D UI illustration mixed with crisp vector-like diagram elements, inspired by modern developer tool product art.
Composition/framing: wide centered composition, enough negative space at top and sides for README layout, clear pipeline from left to right.
Lighting/mood: focused, high-trust, calm, precise, cinematic soft glow.
Color palette: graphite black, off-white, mint green, cobalt blue, small amber accents; avoid purple-dominant gradients and one-note blue palettes.
Materials/textures: glassy UI panels, matte terminal surfaces, subtle grid, clean highlights.
Text: no legible text, no logos, no words, no letters, no numbers.
Constraints: must read as a developer documentation/context pipeline, not a generic AI cloud; no watermark.
```
